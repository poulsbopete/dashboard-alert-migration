# Security Policy

## Reporting A Vulnerability

Do not open a public GitHub issue for potential security vulnerabilities.

Choose **one** reporting path:

- **General vulnerability reporting:** use Elastic’s product security process at
  [https://www.elastic.co/product-security](https://www.elastic.co/product-security).
- **Bug bounty–eligible submissions:** use only
  [https://hackerone.com/elastic](https://hackerone.com/elastic)—do not use the
  product-security link above for bounty intake.

For **non-vulnerability** security inquiries (not suspected vulnerabilities in this
software), contact [security@elastic.co](mailto:security@elastic.co).

## Use This Process For

- credential exposure
- auth or permission flaws
- injection or remote code execution risks
- supply-chain integrity concerns

## Use Regular Issues For

- general bugs
- migration fidelity gaps
- documentation issues
- feature requests
